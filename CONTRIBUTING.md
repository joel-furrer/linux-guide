# Contributing

Thank you for your interest in contributing.

## Branch Naming

Use the following branch naming scheme:

```text
feat/<topic>
fix/<topic>
chore/<topic>
```

Examples:

```text
feat/process-management
feat/git-basics

fix/permissions-example

chore/sidebar-order
```

## Commit Messages

This project uses Conventional Commits.

Examples:

```text
feat(git): add detached HEAD explanation

fix(permissions): correct chmod example

docs(systemd): add service examples

chore(docusaurus): update dependencies
```

## Pull Requests

- One topic per PR.
- Keep changes focused.
- Verify command examples before submitting.
- Follow existing writing style.

## Documentation Style

- Use English.
- Use Markdown.
- Explain concepts before commands.
- Include practical examples.
- Prefer clarity over brevity.

## Review Process

All pull requests require review before merging.
